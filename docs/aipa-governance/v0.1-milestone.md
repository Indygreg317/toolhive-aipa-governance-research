# AIPA MCP Governance Research Overlay v0.1 Milestone

This document marks the v0.1 milestone for the independent AIPA MCP governance research overlay.

## Milestone summary

v0.1 demonstrates that AIPA governance artifacts can describe, review, and audit MCP-related decisions across multiple tool and server lifecycle contexts.

The milestone includes examples for:

- filesystem tool-use governance
- browser/search source review governance
- database read-only query governance
- MCP server install governance
- denied MCP server install governance
- unsupported verification boundary handling

The milestone also demonstrates the full governance outcome set:

```text
PASS
FAIL
UNSUPPORTED
```

## Core thesis

Runtime and security systems can control what an agent is allowed to do.

AIPA governance artifacts can explain why an action or server lifecycle decision was allowed, denied, or left unsupported, under what policy, with what evidence, and within what verification boundary.

## Included artifact types

v0.1 includes examples and supporting documentation for:

- Governance Knowledge Blocks
- server trust profiles
- approval, denial, and escalation policy blocks
- execution receipts
- governance records
- verification boundary maps
- audit package summaries
- scenario folders
- validation manifest
- local validator
- GitHub Actions validation workflow

## Scenario coverage

v0.1 includes these scenario packages:

| Scenario | Expected outcome | Purpose |
| --- | --- | --- |
| Filesystem Write Review | PASS | Demonstrates reviewable governance for a high-risk filesystem write request. |
| Browser Search Source Review | PASS | Demonstrates source identifiers, citation requirements, and unsupported-claim boundaries for browser/search use. |
| Database Read-Only Query Review | PASS | Demonstrates scoped database query governance and result-handling review. |
| MCP Server Install Review | PASS | Demonstrates server lifecycle approval governance. |
| Denied MCP Server Install | FAIL | Demonstrates governance-supported denial when trust context or scope evidence is insufficient. |
| Unsupported Verification Boundary | UNSUPPORTED | Demonstrates evidence-bound review when required evidence is outside the verification boundary. |

## Validator coverage

The v0.1 validator supports structural validation for:

- Governance Knowledge Blocks
- execution receipts
- server trust profiles
- install governance records
- approval, denial, and escalation policy blocks
- runtime tool-use scenario folders
- MCP server install scenario folders
- unsupported verification boundary scenario folders

The validation manifest separates:

```text
structural validation
expected governance outcome
```

This means a structurally valid `FAIL` or `UNSUPPORTED` scenario should not break CI when that outcome is expected.

## Explicit boundaries

v0.1 is a research prototype.

It does not claim:

- official ToolHive or Stacklok integration
- runtime enforcement
- production certification
- cryptographic proof
- external attestation
- live policy engine integration
- live MCP registry integration
- endorsement by any third party

## What v0.1 proves

v0.1 proves that AIPA-style governance artifacts can be organized into reviewable packages around MCP-related decisions.

It also proves that governance review does not need to collapse all outcomes into approval or denial. The overlay can represent:

- evidence-supported approval
- evidence-supported denial
- evidence-insufficient unsupported status

## Next research directions

v0.1 should be treated as a stable research milestone, not a production release.
