# Positioning

AIPA MCP Governance Knowledge Blocks are a complementary governance layer for MCP-based agent workflows.

They describe governance context around tool use. They do not replace MCP runtimes, registries, policy engines, or platform controls.

## Core distinction

MCP platforms can answer operational questions such as what tool is available and what access rules apply.

AIPA governance artifacts answer review questions such as:

- Why was this tool use permitted?
- Which policy was referenced?
- What identity context applied?
- Was human oversight required?
- What evidence should be retained?
- What verification boundary applies?
- Can a reviewer produce PASS, FAIL, or UNSUPPORTED?

## Clean boundary statement

AIPA governance records do not decide whether an agent can use a tool. They describe the governance context around that decision: policy reference, identity context, oversight requirement, evidence requirements, execution boundary, and verification status.

## Partnership-safe framing

Use neutral language such as:

- ToolHive-style MCP registry
- MCP governance platform
- agent-runtime platform
- compatible governance overlay
- complementary open governance layer
- research prototype

Avoid language that implies endorsement, replacement, competition, or official integration unless such a relationship exists.
