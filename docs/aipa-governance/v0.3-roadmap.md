# AIPA MCP Governance Research Overlay v0.3 Roadmap

This roadmap describes the research phase after the v0.2 partner-reviewable milestone.

v0.2 established the AIPA MCP governance overlay as a clear, repeatable, partner-safe reference package.

v0.3 moves the project from:

```text
partner-reviewable governance reference package
```

to:

```text
more testable and reusable governance overlay toolkit
```

The goal is still not runtime integration.

The goal is stronger examples, clearer validation, better template reuse, and more precise evidence handling.

### Priority 3: validator hardening

Completed in:

```text
validator/aipa-governance/validate_governance_blocks.py
validator/aipa-governance/README.md
```

The validator hardening adds structural checks for:

- referenced artifact paths
- optional review handoff packages
- policy fingerprint consistency
- agent and MCP server identity consistency where present
- high-risk human oversight fields
- unsupported claims lists
- expected governance outcome values

The validator remains structural and evidence-bound.

It does not perform runtime enforcement, cryptographic proof, external attestation, or production compliance review.

## v0.3 non-goals

v0.3 does not provide:

- official ToolHive integration
- production runtime enforcement
- live MCP registry integration
- commercial certification
- cryptographic proof system
- external attestation service
- enterprise deployment tooling
- claims of Stacklok endorsement or partnership

## v0.3 success criteria

v0.3 is successful if a reviewer can:

- create a new scenario from a reusable template
- understand how the validator checks structural correctness
- see policy fingerprint consistency in a concrete example
- inspect a review handoff package with less ambiguity
- distinguish reviewable claims from unsupported claims
- understand when this overlay pattern is appropriate for another repository

## Status

v0.3 is implemented as a testable and reusable research milestone.

The repository remains a research overlay only. It does not claim official integration, endorsement, production runtime enforcement, certification, cryptographic proof, or external attestation.
