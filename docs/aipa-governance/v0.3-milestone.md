# AIPA MCP Governance Research Overlay v0.3 Milestone

This document marks the v0.3 research milestone for the AIPA MCP governance overlay.

v0.3 moves the project from a partner-reviewable governance reference package toward a more testable and reusable governance overlay toolkit.

The project remains an independent AIPA research fork.

It does not claim official ToolHive or Stacklok integration, endorsement, certification, runtime enforcement, cryptographic proof, external attestation, or production deployment readiness.

## v0.3 objective

The v0.3 objective was:

```text
make the overlay pattern more testable, reusable, and evidence-bound
```

The milestone strengthens the repository so a reviewer can understand:

- how to create a new scenario from reusable templates
- how a template-instantiated package should look
- how the validator checks structural consistency
- how policy fingerprints evolve from placeholders to canonical candidates
- how review handoff packages should be inspected
- when the overlay pattern is appropriate for another public repository
- what claims must remain unsupported

## What v0.3 added

### Template usage guide

The template usage guide explains how to create a new governance scenario from reusable overlay templates.

It covers:

- choosing the correct template
- creating a scenario folder
- replacing placeholders
- assigning policy references and fingerprints
- keeping fingerprints consistent
- declaring evidence inputs
- separating reviewable and unsupported claims
- choosing `PASS`, `FAIL`, or `UNSUPPORTED`
- creating boundary maps and audit summaries
- adding scenarios to the manifest when appropriate
- running validation

### Template-instantiated example scenario

The template-instantiated example demonstrates how the runtime tool-use overlay template can become a concrete review package.

Scenario path:

```text
examples/aipa-governance/scenarios/template-instantiated-example/
```

The package includes:

- request
- decision context
- execution receipt
- governance record
- verification boundary map
- audit package summary
- review handoff package

### Validator hardening

The validator now performs stronger structural and consistency checks for scenario folders.

It checks items such as:

- referenced repository paths
- optional review handoff packages
- policy fingerprint consistency
- agent and server identity consistency where present
- high-risk human oversight shape
- unsupported claim lists
- expected governance outcome values

The validator remains structural and evidence-bound.

It does not perform runtime enforcement or proof.

### Policy fingerprint example

The policy fingerprint example demonstrates the distinction between:

```text
placeholder fingerprint
```

and:

```text
canonical policy fingerprint candidate
```

Example path:

```text
examples/aipa-governance/policy-fingerprint-example.json
```

The example keeps canonical policy fingerprints explicitly in candidate status until canonicalization and recomputation are implemented.

### Review handoff index

The review handoff index gives external reviewers a concrete inspection path through a handoff package.

It maps:

- handoff package fields
- governance record references
- evidence references
- policy references
- policy fingerprints
- verification boundary maps
- audit package summaries
- unsupported claims
- expected outcomes

### Overlay reuse criteria

The overlay reuse criteria document defines when this pattern is appropriate for future public repositories.

It includes:

- required candidate criteria
- strong and weak candidate signals
- do-not-use rules
- lightweight scoring model
- public-repo candidate worksheet
- partner-safe language guidance
- evidence-bound positioning requirements
