# AIPA MCP Governance Research Overlay v0.2 Milestone

This document marks the v0.2 research milestone for the AIPA MCP governance overlay.

v0.2 moves the project from a credible v0.1 research prototype into a partner-reviewable governance reference package.

The project remains an independent AIPA research fork.

It does not claim official ToolHive or Stacklok integration, endorsement, certification, runtime enforcement, cryptographic proof, or production deployment readiness.

## v0.2 objective

The v0.2 objective was:

```text
clarity, repeatability, reviewability, and partner-safe positioning
```

The milestone strengthens the repository so a reviewer can understand:

- what the project is
- what it is not
- how to inspect the examples
- how to run validation
- how PASS, FAIL, and UNSUPPORTED outcomes are represented
- how governance records differ from independent verification
- how policy fingerprints act as review anchors
- how the overlay pattern can be reused safely

## What v0.2 added

### Reviewer walkthrough

A reviewer-facing walkthrough explains how to inspect the repository quickly, run validation, and understand the scenario packages.

### Architecture diagrams

Mermaid diagrams show the separation between:

```text
runtime / security layer
```

and:

```text
governance / review layer
```

### Partner review package

The partner review package explains the project in neutral, non-competitive language for maintainers, security engineers, governance researchers, and external reviewers.

### Review handoff model

The review handoff model separates:

```text
governance declaration
```

from:

```text
independent review or verification
```

This supports the AIPA boundary principle that governance records organize claims and evidence while review systems evaluate whether the evidence supports the claims.

### Policy fingerprint evolution

The policy fingerprint documentation defines fingerprints as consistency anchors for v0.2.

They help reviewers detect missing, mismatched, or unsupported policy context.

They are not presented as cryptographic proof in this milestone.

### Handoff validation coverage

The validator now recognizes review handoff packages and checks referenced paths and policy fingerprint consistency across the handoff package.

### Reusable overlay templates

The reusable template package provides copyable starter structures for:

- runtime tool-use review
- MCP server install or exposure review
- external review handoff packages
