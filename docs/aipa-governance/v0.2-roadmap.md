# AIPA MCP Governance Research Overlay v0.2 Roadmap

This roadmap describes the research phase after the v0.1 milestone.

v0.1 proved that AIPA governance artifacts can be organized into reviewable scenario packages across `PASS`, `FAIL`, and `UNSUPPORTED` outcomes.

v0.2 moves the project from:

```text
credible research prototype
```

to:

```text
partner-reviewable governance reference package
```

The goal is not runtime integration. The goal is clarity, repeatability, reviewability, and partner-safe positioning.

## v0.2 non-goals

v0.2 does not attempt to provide:

- official ToolHive integration
- production runtime enforcement
- live MCP registry integration
- commercial certification
- cryptographic proof system
- external attestation service
- enterprise deployment tooling

Those may be future research directions, but they are not v0.2 claims.

## v0.2 success criteria

v0.2 is successful if a reviewer can understand:

- what the project is
- what it is not
- how to run validation
- how the scenarios demonstrate governance review
- how AIPA artifacts complement MCP-style runtimes
- how external review could receive evidence without trusting the originating governance layer
- how the overlay pattern could be repeated safely across compatible public repositories

## Status

v0.2 is implemented as a partner-reviewable research milestone.

The repository remains a research overlay only. It does not claim official integration, endorsement, production runtime enforcement, certification, cryptographic proof, or external attestation.
