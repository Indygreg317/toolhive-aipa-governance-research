# AIPA Governance Research Notice

This fork contains an independent AIPA research overlay for MCP governance Knowledge Blocks.

## Purpose

The purpose of this fork is to explore how AIPA governance artifacts can sit beside ToolHive-style MCP workflows without replacing the underlying MCP runtime, registry, policy, or platform controls.

The AIPA overlay focuses on governance context:

- policy references
- policy fingerprints
- identity context
- human oversight requirements
- execution receipts
- governance records
- verification boundary maps
- audit package summaries

## Relationship to ToolHive and Stacklok

This repository is an independent research fork. It does not claim endorsement, approval, partnership, or official integration with Stacklok, ToolHive, or any related project.

The original ToolHive project remains the upstream open-source project. This fork adds a separate research layer under AIPA-specific folders.

## Research thesis

Security and runtime platforms can control what agents are allowed to do.

Governance artifacts can explain why an action was allowed, under what policy, with what oversight, and what evidence exists for later review.
